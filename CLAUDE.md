# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
# Install dependencies (use bun locally)
bun install

# TypeScript check + Vite bundle
bun run build

# Full theme build: TypeScript + Vite + Keycloakify JAR (requires Maven)
bun run build-keycloak-theme

# Storybook hot-reload preview → http://localhost:6006
bun run storybook

# Docker local test → http://localhost:8080 (admin/admin)
docker compose build
docker compose up

# Build with specific Keycloak version
KEYCLOAK_VERSION=26.6.3 docker compose build
```

CI uses `npm` (not bun). Docker build also uses `npm`. Bun is only for local development.

## Architecture

**Keycloakify v11** wraps the React app into a Keycloak theme JAR. The build pipeline is: `tsc -b` → `vite build` → `keycloakify build` → `dist_keycloak/*.jar`.

### Page routing

`src/login/KcPage.tsx` switches on `kcContext.pageId`. Three pages have full custom implementations:
- `login.ftl` → `pages/Login.tsx`
- `register.ftl` → `pages/Register.tsx`
- `login-reset-password.ftl` → `pages/LoginResetPassword.tsx`

All other Keycloak flows fall through to `DefaultPage` from keycloakify with the custom `Template` applied.

### Template

`src/login/Template.tsx` is the shared layout. It:
- Renders the full-page dark glassmorphism panel (420px right-aligned sidebar over `background.png`)
- Injects global CSS via `useEffect` (`<style>` tag appended to `document.head`) — no CSS modules or Tailwind
- Loads Geist font from Google Fonts at runtime

### Design tokens (hardcoded in Template.tsx and page components)

| Token | Value |
|---|---|
| Input background | `#27242E` |
| Input border | `#34343B` |
| Panel background | `rgba(17, 19, 27, 0.96)` |
| Primary gradient | `#6844FF` → `#1D283A` (117deg) |
| Link color | `#6366F1` |
| Error color | `#ff6b7a` |
| Font | Geist (fallback: Open Sans) |

### Generated files

`src/kc.gen.tsx` is auto-generated by keycloakify — do not edit manually. It re-exports `KcContext`, `ThemeName`, `KcEnvName`, and the main entry points.

### Adding a new custom page

1. Create `src/login/pages/MyPage.tsx` implementing `PageProps<Extract<KcContext, { pageId: "my-page.ftl" }>, I18n>`
2. Add a `case "my-page.ftl":` in `KcPage.tsx`
3. Optionally add `MyPage.stories.tsx` using `createKcPageStory({ pageId: "my-page.ftl" })`

### Storybook

Stories live alongside pages (`*.stories.tsx`). `KcPageStory` / `createKcPageStory` from `src/login/KcPageStory.tsx` provides mock `kcContext` fixtures. Use story `args` to override specific `kcContext` fields for state variants.

## Releases
